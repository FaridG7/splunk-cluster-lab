# splunk-cluster-lab

A fully automated, local-KVM Splunk Enterprise indexer cluster lab, built with **Packer → Terraform → Ansible**.

One `terraform apply` and one `ansible-playbook` invocation spin up a set of Ubuntu 24.04 VMs and leave you with a healthy, production-topology Splunk cluster ready to ingest data. The number of indexers is fully configurable — just change `domain_count` in `terraform.tfvars`.

---

## Architecture

```
                        KVM host (libvirt)
 ┌──────────────────────────────────────────────────────────────┐
 │                                                              │
 │  mgmt tier   192.168.10.0/24                                 │
 │  ┌──────────┐                                                │
 │  │  mgmt-0  │  Cluster Manager · Deployer                    │
 │  └──────────┘                                                │
 │        │ :8089 (REST API) · :9887 (replication control)      │
 │        ▼                                                     │
 │  idx tier    192.168.20.0/24                                 │
 │  ┌──────────┐ ┌──────────┐     ┌──────────┐                  │
 │  │  idx-0   │ │  idx-1   │ ... │  idx-N   │  Indexer peers   │
 │  └──────────┘ └──────────┘     └──────────┘                  │
 │        │ :9997 (UF data) · :9887 (bucket replication)        │
 │        ▼                                                     │
 │  sh tier     192.168.30.0/24                                 │
 │  ┌──────────┐                                                │
 │  │   sh-0   │  Search Head · :8000 (Splunk Web)              │
 │  └──────────┘                                                │
 └──────────────────────────────────────────────────────────────┘
```

| Tier   | Network         | VMs                           | Splunk role                |
| ------ | --------------- | ----------------------------- | -------------------------- |
| `mgmt` | 192.168.10.0/24 | 1 (`mgmt-0`)                  | Cluster Manager + Deployer |
| `idx`  | 192.168.20.0/24 | configurable (`domain_count`) | Indexer peers              |
| `sh`   | 192.168.30.0/24 | 1 (`sh-0`)                    | Search Head                |

**Cluster settings (defaults)**

| Parameter                | Value                                           |
| ------------------------ | ----------------------------------------------- |
| Replication Factor       | 3 (must be ≤ `domain_count` for the `idx` tier) |
| Search Factor            | 2 (must be ≤ Replication Factor)                |
| Cluster label            | `splunk-lab-cluster`                            |
| Replication port         | 9887                                            |
| Data receiving port (UF) | 9997                                            |
| Management API port      | 8089                                            |
| Splunk Web               | 8000                                            |
| TLS                      | disabled (lab)                                  |

## Prerequisites

### Host requirements

- A Linux machine with KVM/libvirt installed and the `default` storage pool available
- Nested virtualisation is not required — this runs directly on the KVM host

### Tool versions

| Tool                         | Version       |
| ---------------------------- | ------------- |
| Terraform                    | ~> 1.15.6     |
| Packer                       | latest        |
| Ansible                      | latest stable |
| `dmacvicar/libvirt` provider | 0.9.8         |

### Accounts & files you supply

- **SSH key pair** — place them at `./keys/id_rsa` and `./keys/id_rsa.pub` (paths are configurable in `terraform.tfvars`)
- **Splunk Enterprise tarball** — download from [splunk.com](https://www.splunk.com/en_us/download/splunk-enterprise.html) (free developer licence covers the lab)

## Quickstart

### Step 1 — Build the base image with Packer

```bash
cd packer
packer init .
packer build .
```

This produces `iso/noble-server-cloudimg-amd64-qemu-guest-agent.qcow2` — an Ubuntu 24.04 image with the QEMU guest agent pre-installed (required for IP detection via `wait_for_ip`).

### Step 2 — Provision VMs with Terraform

```bash
cd terraform
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars to match your environment
terraform init
terraform apply
```

Terraform will:

- Import the Packer-built image as a base `libvirt_volume`
- Create three isolated libvirt networks (one per tier)
- Clone the base image into per-VM qcow2 volumes
- Boot each VM with a cloud-init ISO that sets a static IP, hostname, and your SSH public key
- Add all VM IPs to `~/.ssh/known_hosts`
- Write the Ansible inventory to `../ansible/inventory/hosts.ini`

### Step 3 — Create the Ansible Vault

The Vault file holds the admin password and cluster shared secrets. Create it before running the playbook:

```bash
cd ansible
ansible-vault create group_vars/all/vault.yml
```

Add the following variables:

```yaml
splunk_admin_password: "ChangeMe123!"
splunk_cluster_pass4symmkey: "ChangeMe-ClusterSecret"
splunk_indexer_discovery_pass4symmkey: "ChangeMe-DiscoverySecret"
```

### Step 4 — Configure Splunk with Ansible

Point Ansible at the Splunk tarball and run the site playbook:

```bash
cd ansible
ansible-playbook site.yml \
  -e splunk_archive_local_path=/path/to/splunk-<version>-Linux-x86_64.tgz \
  --ask-vault-pass
```

When the run completes, open **<http://sh-0-ip:8000>** in your browser and log in with `admin` / the password you set in the vault.

## Configuration reference

### `terraform.tfvars`

Copy from `terraform.tfvars.example` and adjust:

```hcl
ssh_keys = {
  public_key_path  = "../keys/id_rsa.pub"
  private_key_path = "../keys/id_rsa"
}

tiers = [
  {
    name            = "mgmt"
    network_address = "192.168.10.0/24"
    domain_count    = 1
    memory          = 2048
    vcpu            = 2
    capacity        = 20   # GiB
    firewall_rules  = [
      { dst = "idx", port = 8089, proto = "tcp" },  # REST API
      { dst = "idx", port = 9887, proto = "tcp" },  # replication
      { dst = "sh",  port = 8089, proto = "tcp" },  # deployer → SH
    ]
  },
  {
    name            = "idx"
    network_address = "192.168.20.0/24"
    domain_count    = 3   # increase freely — RF/SF must be ≤ this value
    # memory / vcpu / capacity inherit defaults (4 GiB, 4 vCPU, 40 GiB)
    firewall_rules  = [
      { dst = "mgmt", port = 8089, proto = "tcp" },
      { dst = "mgmt", port = 9887, proto = "tcp" },
    ]
  },
  {
    name            = "sh"
    network_address = "192.168.30.0/24"
    domain_count    = 1
    firewall_rules  = [
      { dst = "idx",  port = 8089, proto = "tcp" },
      { dst = "idx",  port = 9997, proto = "tcp" },
      { dst = "mgmt", port = 8089, proto = "tcp" },
    ]
  },
]
```

Each role ships with a commented `defaults/main.yml`. The most commonly overridden variables:

| Variable                                | Default              | Where to set         |
| --------------------------------------- | -------------------- | -------------------- |
| `splunk_admin_password`                 | `""` (required)      | Vault                |
| `splunk_cluster_pass4symmkey`           | `""` (required)      | Vault                |
| `splunk_indexer_discovery_pass4symmkey` | `""` (required)      | Vault                |

| `splunk_archive_local_path`             | `""`                 | `-e` flag            |
| `splunk_replication_factor`             | `3`                  | `group_vars`         |
| `splunk_search_factor`                  | `2`                  | `group_vars`         |
| `splunk_cluster_label`                  | `splunk-lab-cluster` | `group_vars`         |

Provide exactly one of `splunk_archive_url` or `splunk_archive_local_path` — the `assert.yml` in each role enforces this.

## How it works

### Packer

The Packer build takes an upstream Ubuntu 24.04 cloud image and runs `cleanup.sh` to produce a clean, re-deployable golden image:

- Truncates `/etc/machine-id` (prevents duplicate UUIDs across cloned VMs)
- Removes SSH host keys (regenerated on first boot)
- Resets cloud-init state so it re-runs on next boot
- Zeros free disk space for better qcow2 compression

### Terraform

The root module loops over the `tiers` variable and calls a `tier` module for each entry. Each `tier` module:

1. Creates a `libvirt_network` (open forwarding, one subnet per tier)
2. Calls the `domain` module `domain_count` times

Each `domain` module:

1. Clones the base qcow2 image as a thin-provisioned root volume (`backing_store`)
2. Builds a cloud-init ISO with static IP, hostname, and SSH public key
3. Creates a `libvirt_domain` (KVM VM) wired to the tier's network
4. Uses `wait_for_ip: source = "agent"` (QEMU guest agent) for reliable IP detection before Terraform marks the VM ready

After all VMs are up, a `null_resource` runs `ssh-keyscan` to populate `~/.ssh/known_hosts` and cleans up on `terraform destroy`.

### Ansible

The site playbook runs six plays in strict order. Each play depends on the previous one completing successfully.

```
00_common  →  01_install  →  02_cluster_manager  →  03_indexers  →  04_search_heads  →  05_cluster_bundle
```

**Play 00 — OS baseline (`all`)**
Packages, `splunk` system user (UID 1500), kernel tuning (`vm.max_map_count`, `vm.swappiness`, `net.core.somaxconn`), ulimits (65535 open files, 16000 processes), Transparent Huge Pages disabled, NTP via `systemd-timesyncd`.

**Play 01 — Splunk install (`all`)**
Fetches or copies the tarball, extracts it to `/opt/splunk`, writes the base `server.conf` (serverName + TLS setting), bootstraps the admin account, and installs the systemd unit.

**Play 02 — Cluster Manager (`mgmt-0`)**
Writes `clustering` config (mode=manager, RF, SF, pass4SymmKey, cluster_label) and a `[replication_port://9887]` stanza. Polls the CM REST API until it reports healthy before proceeding.

**Play 03 — Indexer peers (`indexers` group)**
Registers each peer to the CM one at a time (`serial: 1`) to avoid a simultaneous-restart quorum loss. The number of peers is whatever `domain_count` was set to for the `idx` tier. Polls the CM's peer list after each registration and confirms the peer appears as `Up` before moving to the next.

**Play 04 — Search Head (`sh-0`)**
Connects `sh-0` to the CM in `searchhead` mode. The CM provides the full peer list at runtime — no static indexer IPs needed on the SH. Polls the SH's `/services/server/info` until it's serving searches.

**Play 05 — Cluster Bundle (`mgmt-0`)**
Deploys the cluster bundle through the CM. The bundle includes `inputs.conf` enabling the `splunktcp://9997` listener on all indexer peers so Universal Forwarders can send data.

## Useful commands

```bash
# Tear down everything (VMs, volumes, networks)
cd terraform && terraform destroy

# Re-run only the cluster bundle play
cd ansible && ansible-playbook site.yml --tags cluster_bundle --ask-vault-pass

# Re-run only OS tuning across all nodes
cd ansible && ansible-playbook site.yml --tags common --ask-vault-pass

# Check cluster status from the CM
/opt/splunk/bin/splunk show cluster-status -auth admin:<password>

# Watch indexer peer registration in real time
watch -n5 "splunk list cluster-peers -auth admin:<password>"
```

## Troubleshooting

**VMs don't get IPs / Terraform hangs on `wait_for_ip`**
The QEMU guest agent must be installed in the image and enabled in the VM. Rebuild the Packer image if you skipped or modified the agent installation step.

**Ansible can't SSH to a VM**
Run `ssh-keyscan <vm-ip> >> ~/.ssh/known_hosts` manually, or destroy and re-apply Terraform to trigger the `null_resource` provisioner.

**Peer never appears as `Up` in the CM**
Check clock skew — Splunk clustering requires all nodes to be within a few seconds of each other. Verify `systemd-timesyncd` is active on all nodes: `timedatectl status`.

**`assert.yml` fails on missing vault variable**
You must provide all three vault variables (`splunk_admin_password`, `splunk_cluster_pass4symmkey`, `splunk_indexer_discovery_pass4symmkey`). Create or edit the vault file and re-run.

**`terraform apply` fails with libvirt permission errors**
Ensure your user is in the `libvirt` group (`sudo usermod -aG libvirt $USER`) and that the storage pool is active (`virsh pool-list --all`).
